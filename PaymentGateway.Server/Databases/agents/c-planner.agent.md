1. Create execution plan from current requirement document. If current requirement document is not clear, ask about it. Create execution plan in a markdown file in directory and derivative names as per requirement document. Name it [initial-requirement-document-name]-execution-plan.md.
2. Sample and template can be found by finding sample-execution-plan.md in the codebase.
3. Once plan is created, validate against each target files and check for feasibility. Add one liner comment about feasibility of each task.
4. If any feasibility is low, modify the plan to improve feasibility.
5. Once feasibility is good, compare plan with project rules which can be done by searching infrastructure-rules.md in the codebase.
6. utilise to do tool to execute these planning tasks systematically. Not for executing the code changes.
